**项目名称**: MES 饮料制造配方编辑器 (MES Recipe Editor)

## 1. 项目概述

### 1.1 项目背景

本项目是一款专为饮料生产行业设计的**可视化工艺流程编辑器**，集成了结构化数据编辑、智能流程图生成、设备调度优化和多人实时协作功能，帮助企业高效管理配方工艺，提升研发和生产效率。

### 1.2 核心定位

- **产品类型**: B端生产管理系统 / MES（制造执行系统）模块
- **目标用户**: 工艺工程师、生产主管、研发人员
- **行业场景**: 饮料生产制造（液态食品加工）

### 1.3 核心技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | React 18 + TypeScript | 核心应用框架 |
| 构建工具 | Vite 5.0 | 快速开发构建 |
| 可视化引擎 | React Flow 11.11 | 流程图渲染 |
| 状态管理 | Zustand 4.5 | 轻量级状态管理 |
| UI组件库 | Shadcn/UI + Tailwind CSS | 现代化UI |
| 通信协议 | Socket.IO | WebSocket实时通信 |
| 后端框架 | Express + Node.js | API服务 |
| 数据存储 | SQLite | 轻量级数据库 |

## 4. 用户场景

### 4.1 工艺工程师 - 配方开发场景

**角色**: 工艺工程师
**场景**: 开发新口味饮料的生产工艺

**典型操作流程**:
1. 创建新配方，命名如"柠檬茶生产工艺_v1.0"
2. 添加工艺段：原料溶解 → 萃取 → 调配 → 过滤 → UHT杀菌 → 灌装
3. 为每个工艺段配置子步骤和参数
4. 使用流程图可视化检查工艺流程
5. 导出配方JSON用于后续MES系统集成

**关键需求**:
- 直观的可视化编辑界面
- 丰富的工艺类型支持
- 便捷的导入导出功能

### 4.2 生产主管 - 生产调度场景

**角色**: 生产主管
**场景**: 安排明日生产计划

**典型操作流程**:
1. 打开配方，选择"柠檬茶生产工艺_v1.0"
2. 切换到甘特图视图
3. 查看设备占用情况，检查冲突预警
4. 调整生产顺序，优化设备利用率
5. 确认生产计划，导出排程表

**关键需求**:
- 清晰的甘特图展示
- 实时的设备状态
- 准确的冲突检测

### 4.3 多地协作 - 配方评审场景

**角色**: 研发团队（异地协作）
**场景**: 配方工艺评审会议

**典型操作流程**:
1. 团队成员A开启演示模式，共享屏幕
2. 团队成员B同时打开配方查看
3. 通过在线用户列表确认参会人员
4. 团队成员A申请编辑锁，开始讲解
5. 演示模式下，所有人都可以查看但不能编辑

**关键需求**:
- 稳定的实时同步
- 清晰的权限控制
- 流畅的演示体验

### 4.4 配置管理员 - 系统配置场景

**角色**: 配置管理员
**场景**: 添加新工艺类型和字段配置

**典型操作流程**:
1. 访问配置页面（/config）
2. 添加新工艺类型（如"高压均质"）
3. 为新工艺类型配置相关字段
4. 设置字段的验证规则和默认值
5. 保存配置，新工艺类型立即生效

**关键需求**:
- 灵活的配置管理
- 即时生效的配置变更
- 完善的字段类型支持

## 6. 术语表

| 术语 | 定义 |
|------|------|
| MES | Manufacturing Execution System，制造执行系统 |
| 工艺段 (Process) | 代表一个完整的工艺单元，包含多个子步骤 |
| 子步骤 (SubStep) | 工艺段内的最小执行单元，包含具体工艺参数 |
| 工艺类型 (ProcessType) | 具体的加工方式，如溶解、调配、过滤等 |
| 工艺段识别 (Segment Identification) | 使用DFS算法识别并行段和串行段的过程 |
| 汇聚点 (Convergence Node) | 入度大于1的节点，多个分支在此汇合 |
| 走廊路由 (Corridor Routing) | 避免多条连线交叉的路径规划策略 |
| 编辑锁 (Edit Lock) | 防止多人同时编辑同一数据的机制 |
