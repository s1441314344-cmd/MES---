## 1. 需求概述

### 1.1 项目背景

基于V1版本的BPMN工作流系统，进行重大交互体验优化。参考飞书多维表格工作流的交互模式，移除传统的节点库和手动连线方式，采用更加直观的内容选择器和自动连线机制，大幅提升用户编辑效率和体验。

### 1.2 核心目标

| 目标编号 | 目标描述 | 优先级 |
|---------|---------|--------|
| GOAL-001 | 移除左侧节点库面板，采用内容选择器添加节点 | Must Have |
| GOAL-002 | 取消手动连线，实现节点间自动连线功能 | Must Have |
| GOAL-003 | 完全参考飞书多维表格工作流的交互模式和视觉设计 | Must Have |

### 1.3 飞书工作流设计特征

基于飞书多维表格工作流的典型设计模式：

- **垂直列表布局**：节点以垂直列表形式排列
- **节点间插入点**：节点之间有"+"按钮，用于插入新节点
- **自动连线**：节点按顺序自动连接，无需手动拖拽连线
- **右侧属性面板**：选中节点后右侧显示详细属性配置
- **简洁卡片设计**：每个节点是一个卡片，包含图标、标题、状态
- **无画布拖拽**：节点位置固定，通过添加/删除调整顺序

#### 4.2.1 简化的节点结构

```typescript

  id: string;
  type: NodeType;
  order: number;  // 新增：用于排序
  data: NodeData;
}

  name: string;
  description?: string;

```

#### 4.2.2 简化的连线结构

```typescript
interface WorkflowEdge {
  id: string;
  source: string;
  target: string;
  type: 'sequence' | 'conditional';
}
```

### 4.3 核心算法

#### 4.3.1 自动连线算法

```
输入：节点列表（按order排序）
输出：连线列表

1. 遍历节点列表（索引i从0到n-2）
2.   当前节点 = nodes[i]
3.   下一节点 = nodes[i+1]
4.   创建连线：source=当前节点.id, target=下一节点.id
5. 返回连线列表
```

#### 4.3.2 插入节点算法

```
输入：节点列表、新节点、插入位置index
输出：更新后的节点列表

1. 新节点.order = index
2. 原index及之后的节点.order += 1
3. 将新节点插入到index位置
4. 重新计算所有连线
5. 返回更新后的列表
```

#### 4.3.3 删除节点算法

```
输入：节点列表、待删除节点ID
输出：更新后的节点列表

1. 找到待删除节点的索引index
2. 删除该节点
3. index之后的节点.order -= 1
4. 重新计算所有连线
5. 返回更新后的列表
```

## 5. 验收标准

### 5.1 功能验收

#### AC-001: 垂直列表布局

**EARS格式:**
```
WHEN 用户打开工作流编辑器，
THEN 系统应显示垂直排列的节点列表，
AND 每个节点显示为圆角卡片。
```

#### AC-002: 内容选择器添加节点

**EARS格式:**
```
WHEN 用户点击节点间的"+"按钮，
THEN 系统应弹出内容选择器，
WHEN 用户选择一个节点类型，
THEN 系统应在该位置添加新节点，
AND 自动更新连线关系。
```

#### AC-003: 自动连线

**EARS格式:**
```
WHEN 节点列表发生变化（添加/删除），
THEN 系统应自动重新计算连线关系，
AND 确保节点按顺序正确连接。
```

#### AC-004: 飞书风格交互

**EARS格式:**
```
WHEN 用户使用编辑器，
THEN 整体交互体验应与飞书多维表格工作流保持一致，
包括视觉风格、操作流程、反馈机制。
```

### 5.2 用户体验验收

- [ ] 新用户3分钟内可完成基础流程创建
- [ ] 所有操作有明确的视觉反馈
- [ ] 界面简洁直观，无学习障碍
- [ ] 响应时间 < 200ms
